### E. 增加“体验与伦理约束”

- 连续失败时只给温和鼓励，不惩罚

### F. 增加“可测试标准”

- Given/When/Then 场景
- 空状态、异常状态、离线状态

## 3) “沉浸式专注陪伴工具”完整产品方案（可直接立项）

## 3.1 产品愿景与定位
- 愿景：让用户在“被陪伴”的情绪安全感中，稳定完成日常专注任务。
- 定位：专注效率工具 + 轻陪伴养成，不做重剧情游戏。
- 差异点：角色互动与专注行为强绑定，互动发生在休息窗口，不打断深度专注。

## 3.2 目标用户与核心场景

### 用户分层
1. 学生党：备考、自习、论文冲刺。
2. 初中级职场：写方案、复盘、学习技能。
3. 自由职业者：任务碎片化、缺少外部约束。

### 核心场景
- 晚间独自学习 2 小时，需要环境音 + 陪伴感。
- 白天工作被消息打断，需要深度模式屏蔽干扰。
- 想长期养成“每日至少 2 个番茄”的习惯。

## 3.3 功能设计

### 1) 专注系统（核心闭环）
- 计时参数：25/45/60（可自定义）。
- 休息参数：5/10（可自定义）。
- 模式：
  - 标准模式：提醒正常可见。
  - 深度模式：屏蔽角色弹窗，仅保留结束提示。
- 结果判定：完成/中断/超时结束。
- 奖励：积分 + 好感 + 连续计数。

**MVP 边界**
- 单设备本地通知 + 云同步记录。

**二期扩展**
- 智能建议时长（基于历史专注成功率）。

### 2) 音乐与环境系统
- 音轨池：白噪音、自然声、Lo-fi。
- 混音台：主音量、环境音量、音乐音量。
- 预设：保存“雨夜学习/咖啡馆办公”等组合。

**MVP 边界**
- 内置 10 条可循环音轨 + 5 个预设模板。

### 3) 虚拟人物 A（核心亮点）
- 常驻位置：主界面背景或侧边角色位。
- 触发时机：
  - 专注开始：鼓励 1 句。
  - 休息开始：主动发起 1 轮对话。
  - 完成里程碑：特殊祝贺动画。
- 好感等级：Lv1-Lv10。
- 性格层次：
  - 低等级：礼貌鼓励型。
  - 中等级：理解与共情。
  - 高等级：目标导向 + 关怀型。

**互动约束**
- 不在专注计时中弹出多轮对话。
- 用户可一键“静默角色 24h”。

### 4) 笔记系统
- 快速输入入口（悬浮按钮 / 快捷键）。
- 自动挂载到当前番茄记录。
- 标签体系：任务标签、情绪标签、主题标签。
- 检索：关键词 + 标签 + 日期范围。

### 5) 习惯追踪系统
- 目标粒度：每日番茄数、每周总时长。
- 看板：日/周/月热力图。
- 机制：连续打卡、断签恢复卡（每月 1 次）。

## 3.4 养成数值系统（首版建议）

### 基础公式
- `好感 = floor(专注分钟/25) * 10 * 连续加成 * 模式系数`
- 连续加成：
  - 连续 1-3 个番茄：1.0
  - 第 4-6 个：1.2
  - 7 个及以上：1.3
- 模式系数：
  - 标准：1.0
  - 深度：1.1

### 抑制刷分
- 每日好感上限 200。
- 单日同一时长重复快速完成触发衰减（第 6 次后 *0.8）。

### 等级区间（示例）
- Lv1: 0-99
- Lv2: 100-249
- Lv3: 250-449
- Lv4: 450-699
- Lv5: 700-999
- Lv6: 1000-1399
- Lv7: 1400-1899
- Lv8: 1900-2499
- Lv9: 2500-3199
- Lv10: 3200+

## 3.5 休息阶段互动机制
- 输入：当前等级、最近 3 次专注状态、时间段（早/午/晚）、用户心情标签。
- 输出：1 条主动语句 + 2-3 个可选回应。
- 选项影响：
  - “聊目标” -> 提升任务导向人格值
  - “聊感受” -> 提升关怀人格值
- 高好感解锁：
  - 周总结对话
  - 节日特别台词
  - 连续 7 天剧情片段

## 3.6 信息架构（IA）
- 首页（计时器 + 环境 + 角色）
- 专注记录页（时间轴、完成率）
- 笔记页（列表/详情/标签）
- 习惯页（目标、连续天数、热力图）
- 角色页（等级、剧情、互动偏好）
- 设置页（提醒、隐私、音频、静默）

## 3.7 关键用户流程
1. 新用户 3 分钟引导：选目标 -> 选番茄时长 -> 选声音预设 -> 开始首个番茄。
2. 专注流程：开始 -> 计时 -> 完成 -> 奖励反馈 -> 休息互动 -> 下一轮。
3. 复盘流程：查看今日时长 -> 打标签总结 -> 更新明日目标。

## 3.8 数据模型（MVP）

### 核心表
- `users`：用户资料、时区、偏好。
- `focus_sessions`：每次番茄记录（开始/结束/状态/模式/时长）。
- `audio_presets`：用户音频预设。
- `notes`：笔记内容与关联 session_id。
- `habits`：习惯目标定义。
- `habit_checkins`：每日打卡。
- `companion_profile`：角色等级、好感、人格倾向。
- `companion_events`：剧情解锁与互动日志。

### 关键字段建议
- `focus_sessions.status`: completed/interrupted/abandoned
- `focus_sessions.interrupt_seconds`
- `companion_profile.affection_total`
- `companion_profile.persona_care` / `persona_goal`

## 3.9 API 草案（示例）
- `POST /api/focus/start`
- `POST /api/focus/finish`
- `POST /api/focus/interrupt`
- `GET /api/focus/stats?range=week`
- `POST /api/notes`
- `GET /api/notes?tag=&q=`
- `POST /api/companion/respond`
- `GET /api/companion/profile`
- `POST /api/habits/checkin`

## 3.10 埋点与指标

### 北极星指标
- 周有效专注总分钟数（WAFT: Weekly Active Focus Time）

### 核心埋点（10 条）
1. `focus_start`
2. `focus_complete`
3. `focus_interrupt`
4. `rest_dialog_open`
5. `rest_dialog_option_click`
6. `audio_preset_apply`
7. `note_create`
8. `habit_checkin`
9. `story_unlock`
10. `deep_mode_enable`

### 关键漏斗
- 启动 App -> 开始番茄 -> 完成首个番茄 -> 7 日内完成 10 个番茄。

## 3.11 商业化与版本策略
- 免费层：基础番茄、基础音轨、Lv1-Lv3 陪伴。
- 订阅层：高级音轨、角色主题皮肤、完整剧情线、深度周报。
- 付费原则：不卖“效率焦虑”，只卖“个性化体验”。

## 3.12 风险与合规
- 隐私：笔记默认私密，支持本地导出与账户删除。
- 情感边界：角色文案避免依赖暗示。
- 防沉迷：每日互动上限与夜间温和提醒。
- 内容安全：对话内容策略过滤（低俗、攻击、自伤风险识别）。

## 3.13 MVP 四周计划
- 第 1 周：信息架构、UI 线框、计时与音频 MVP。
- 第 2 周：专注记录 + 笔记 + 习惯打卡。
- 第 3 周：角色好感系统 + 休息对话 + 剧情解锁。
- 第 4 周：埋点、性能优化、灰度发布与复盘。

## 3.14 验收标准（节选）
1. Given 用户设置 25/5，When 完成计时，Then 记录状态为 completed 且奖励正确入账。
2. Given 用户在第 10 分钟离开超过 3 分钟，When 返回，Then 本次记录为 interrupted 且仅发放 20% 积分。
3. Given 用户连续 7 天每天至少 1 个番茄，When 第 7 天完成后，Then 解锁特殊剧情并记录 `story_unlock` 事件。
